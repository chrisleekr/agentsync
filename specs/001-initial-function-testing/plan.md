# Implementation Plan: Initial Function Testing

**Branch**: `001-initial-function-testing` | **Date**: 2026-04-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-initial-function-testing/spec.md`

## Summary

Add comprehensive test coverage to every exported function in the `agent-sync` codebase. The project has never been tested beyond two thin smoke-test files (`schema.test.ts`, `sanitizer.test.ts`). The implementation adds `*.test.ts` files co-located with each module, using the `bun:test` runner, to verify: encryption correctness, secret redaction, archive round-trips and zip-slip rejection, config loading, all five agent snapshot/apply pairs, IPC server/client communication, debounced file watching, and the full CLI command path via a local bare git repository. All test isolation uses per-test `mkdtemp` directories with `afterEach` cleanup; no real agent config directories or OS services are touched.

> **Spec assumption correction**: The original spec's Assumptions section referenced "Vitest". The project's constitution §II and existing test files (`schema.test.ts`, `sanitizer.test.ts`) both confirm the sole test runner is `bun test`. This plan uses `bun test` exclusively.

## Technical Context

**Language/Version**: TypeScript 5.8+ (strict mode; `any` forbidden — use `unknown` + Zod)
**Primary Dependencies**: `bun:test` (built-in), `age-encryption ^0.3.0`, `simple-git ^3.27.0`, `tar ^7.4.0`, `zod ^3.23.8`, `@iarna/toml ^2.2.5`
**Storage**: Filesystem (per-test `mkdtemp` directories, removed in `afterEach`)
**Testing**: `bun test` / `bun test --coverage`
**Target Platform**: macOS (primary), Linux, Windows (path-logic only; installer tests mock OS calls)
**Project Type**: CLI tool test suite — no new runtime code or external APIs added
**Performance Goals**: Full test suite completes in < 60 s on a standard developer machine
**Constraints**: No network access; no OS service modification; no reads from the CI user's real `~/.config/agentsync`; throwaway generated keys only (no hard-coded private key material per constitution §I)
**Scale/Scope**: 19 `*.test.ts` files (17 new, 2 extended) covering ~50 exported functions across 21 source modules
